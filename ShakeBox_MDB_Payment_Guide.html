1. 개요

본 문서는 MDB (Multi-Drop Bus) 프로토콜 기반 캐시리스 카드 리더 통신을 위한 MdbCashless Android SDK의 결제 가이드입니다. MDB-RS232 브릿지를 통해 Android 앱에서 카드리더를 제어하며, gs805serial 패키지에 포함되어 있습니다.

1.1 MDB 통신 구조

Android 앱 ←(USB RS232 9600-8N1)→ MDB-RS232 브릿지 ←(MDB bus)→ Cashless Reader
중요: MDB는 GS805 머신과 별도 시리얼 포트를 사용합니다. GS805 제어와 MDB 카드리더 통신은 각각 독립된 USB 시리얼 연결이 필요합니다.

통신 규칙

  • PC → 브릿지: HEX 데이터
  • 브릿지 → PC: ASCII 데이터
  • POLL은 브릿지가 자동 처리
  • 9600 baud, 8N1

1.2 카드 리더 상태 머신

MDB 캐시리스 카드 리더는 6가지 상태를 가집니다.

#상태설명전환 조건
1Inactive전원 ON/Reset 후 초기 상태전원 ON, Reset
2DisabledConfig 수신 완료Config 응답 수신
3Enabled카드 감지 대기 중Enable 명령 전송
4Session Idle유효 카드 감지, Vend Request 대기유효 카드 감지
5Vend RequestedVMC가 Vend Request 전송requestVend() 호출
6Vending결제 승인, 상품 배출 중Vend 승인 수신
Inactive → Disabled → Enabled → Session Idle → Vend Requested → Vending │ ↑ │ │ │ └──────────┘ (세션 완료/취소) │ │ ↑ │ │ └───────────────────────────────────────────────────┘ │ (vendSuccess/vendCancel) └── setup() ──→ Disabled

2. 설치 및 설정

MdbCashless는 gs805serial 패키지에 포함되어 있습니다. 별도 패키지 설치 없이 동일한 Gradle 의존성을 사용합니다.

Gradle 의존성 추가 (JitPack)

// settings.gradle.kts
dependencyResolutionManagement {
    repositories {
        maven("https://jitpack.io")
    }
}

// build.gradle.kts
dependencies {
    implementation("com.github.anyeats:anyeats-mip-sdk-java:1.0.0")
}

Import

import kr.co.anyeats.gs805serial.mdb.*
// MdbCashless, CashlessEvent, CashlessState 등 포함
참고: MDB 클래스(MdbCashless, CashlessEvent, CashlessState 등)는 kr.co.anyeats.gs805serial.mdb 패키지에 포함되어 있습니다.

4. 결제 플로우

4.1 기본 결제 플로우

val mdb = MdbCashless()

// 1. MDB 브릿지 연결 (GS805와 별도 포트)
val devices = mdb.listDevices()
mdb.connect(devices.first())

// 2. 초기 설정 (앱 시작 시 1회)
mdb.setup(maxPrice = 0xFFFF, minPrice = 0x0000)
mdb.enable()

// 3. 이벤트 수신 (Coroutine scope 내)
mdb.eventFlow.collect { event ->
    when (event.type) {
        CashlessEventType.CARD_DETECTED -> {
            // 카드 감지 → 결제 요청
            val funds = event.data["funds"] as? Int
            println("카드 감지, 잔액: $funds")
            mdb.requestVend(price = 1000, itemNumber = 1)
        }
        CashlessEventType.VEND_APPROVED -> {
            // 결제 승인 → 상품 배출
            val amount = event.data["amount"] as? Int
            println("결제 승인: $amount")
            // ... 상품 배출 로직 ...
            mdb.vendSuccess(itemNumber = 1)
            mdb.sessionComplete()
        }
        CashlessEventType.VEND_DENIED -> {
            println("결제 거부")
            mdb.sessionComplete()
        }
        else -> {}
    }
}

// 4. 종료
mdb.disconnect()
mdb.dispose()

4.2 결제 취소 플로우

// 결제 진행 중 취소
mdb.vendCancel()
mdb.sessionComplete()

4.3 GS805 연동: 카드결제 → 음료제조

// GS805 머신 + MDB 카드리더 동시 운영
val gs805 = GS805Serial(context)
val mdb = MdbCashless()

// 각각 별도 시리얼 포트에 연결
val machineDevices = gs805.listDevices()
gs805.connect(machineDevices.first())

val mdbDevices = mdb.listDevices()
mdb.connect(mdbBridgeDevice)

// MDB 초기화
mdb.setup()
mdb.enable()

// 결제 → 음료 제조 통합 플로우
launch {
    mdb.eventFlow.collect { event ->
        when (event.type) {
            CashlessEventType.CARD_DETECTED -> {
                mdb.requestVend(price = drinkPrice, itemNumber = drinkNumber)
            }
            CashlessEventType.VEND_APPROVED -> {
                gs805.makeDrink(DrinkNumber.HOT_DRINK_1)
            }
            CashlessEventType.VEND_DENIED -> {
                mdb.sessionComplete()
            }
            else -> {}
        }
    }
}

// 음료 완성 이벤트 → 결제 확정
launch {
    gs805.eventFlow.collect { event ->
        if (event.type == MachineEventType.DRINK_COMPLETE) {
            mdb.vendSuccess(itemNumber = 1)
            mdb.sessionComplete()
        }
    }
}
참고: GS805 머신과 MDB 카드리더는 별도의 USB 시리얼 포트를 사용합니다. 연결 시 디바이스의 VID/PID로 구분하여 올바른 장치에 연결해야 합니다.
주의: 결제 승인 후 음료 제조에 실패한 경우 반드시 vendCancel() → sessionComplete()를 호출하여 결제를 취소해야 합니다.

5. 프로토콜 상세

5.1 명령어 목록

VMC → Card Reader (송신)

명령HEX 데이터설명
Config110001000000카드 리더 설정
Set Max/Min Price1101FFFF0000가격 범위 설정
Enable1401활성화
Disable1400비활성화
Cancel1402취소
Request Vend1300{PRICE}{ITEM}결제 요청
Vend Success1302{ITEM}배출 성공
Vend Cancel1301벤딩 취소
Cash Sale1305{PRICE}{ITEM}현금 판매
Session Complete1304세션 종료
Request ID1700리더 ID 요청

Card Reader → VMC (수신)

수신 데이터설명
00ACK
01...Config 응답
03{FUNDS}카드 감지 + 잔액
05{AMOUNT}Vend 승인
06Vend 거부
07Session Complete ACK
04세션 취소

5.2 데이터 형식

Request Vend 명령의 금액 및 상품번호는 2바이트 Big Endian HEX로 인코딩됩니다.

Request Vend 데이터 예시

금액상품번호HEX 데이터
1011300000A0001
1002130000640002
5003130001F40003
10001130003E80001
참고: 금액은 10진수를 16진수로 변환하여 4자리(2바이트)로 표현합니다. 예: 1000 → 0x03E8 → 03E8

6. 에러 처리

상황처리 방법
명령 응답 타임아웃100~200ms 내 응답이 없으면 재전송
카드 리더 비활성카드 리더 전원 및 MDB 연결 상태 확인
결제 승인 후 배출 실패반드시 vendCancel() 호출하여 결제 취소
세션 타임아웃30초 내 세션 완료하지 않으면 자동 취소
 
// 에러 이벤트 처리 예시
mdb.eventFlow.collect { event ->
    if (event.type == CashlessEventType.ERROR) {
        println("MDB Error: ${event.data["error"]}")
    }

    if (event.type == CashlessEventType.VEND_DENIED) {
        // 결제 거부 시 세션 종료
        mdb.sessionComplete()
    }
}

// 음료 제조 실패 시 결제 취소
try {
    gs805.makeDrink(DrinkNumber.HOT_DRINK_1)
} catch (e: Exception) {
    // 결제 승인 상태에서 음료 제조 실패 → 반드시 취소
    mdb.vendCancel()
    mdb.sessionComplete()
}
주의: 결제 승인(VEND_APPROVED) 후에는 반드시 vendSuccess() 또는 vendCancel()을 호출해야 합니다. 호출하지 않으면 카드리더가 Vending 상태에 머물게 됩니다.
